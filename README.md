## 🚀 How to Test the API (Nova Poshta API)

You can run and test our endpoints completely free of charge using any tool you prefer. No paid subscriptions or cloud limits required:

### Option 1. Via Postman 🚀 (One-Click Import)
If you prefer Postman, click the button below to automatically import a copy of the collection into your free account:

[![Run in Postman](https://run.pstmn.io/button.svg)](https://god.gw.postman.com/run-collection/https://web.postman.co/workspace/My-Workspace~1c90ee6a-3800-42cf-8ccd-5f0cbcb19ddb/collection/32325810-83af8aa9-2bf3-42b0-ab00-f6ad231895d8?action=share&source=copy-link&creator=32325810)

*Alternatively, you can download the `api-collection.postman_collection.json` file from this repository and manually drag-and-drop it into Postman via the "Import" button.*

---

### Option 2. Via Bruno 🌿 (Open-Source & Local via YAML)
We have also saved the collection using the modern **OpenCollection YAML standard**. This allows you to track API changes in Git and run requests locally without any cloud synchronization limits.

1. Download and install the free, open-source [Bruno](https://www.usebruno.com/) app.
2. Clone this repository to your local machine.
3. On the Bruno home screen, click **Open Collection**.
4. Select the `bruno-collection/` folder (the app will automatically detect the YAML configuration files inside).
5. You're all set! You can now execute and modify requests locally.
